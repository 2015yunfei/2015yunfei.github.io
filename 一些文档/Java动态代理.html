静态代理

对于典型的静态代理，“静态”体现在代理方法是直接编码在类中的。

 

从实际编码的角度来说，假设遇到了这样一个需求，需要记录下一些方法的执行时间，于是最简单的方式当然就是在方法的开头记录一个时间戳，在 return 之前记录一个时间戳。但如果方法的流程很复杂，例如：

但是还是存在问题：

  1. 如果该方法在很多地方调用，比如还存在Invoker1、Invoker2等等

  2. 需要记录的方法有多个，比如一个Invoker中还有invoke1、invoke2等等

于是可以考虑创建一个代理对象，让它负责帮统一记录时间戳，例如：

静态代理还是会遇到一些问题，如果 Executor 新增了一个方法，同样要记录时间，那就不得不修改 Proxy 的代码。并且如果其他类也有同样的需求，那就需要新建不同的 Proxy 类才能较好的实现该功能，同样非常麻烦。

 

动态代理

概念

Java 动态代理机制是 Java 提供的一种实现代理设计模式的方式，通过它可以在运行时创建代理对象。这种机制允许在不改变原有代码的情况下增强某些功能，比如日志记录、事务处理等。

本文主要是展示 java 动态代理的使用方式和应用场景。

 

在设计模式中有一个非常常用的模式：代理模式。学术一些来讲，就是为某些对象的某种行为提供一个代理对象，并由代理对象完全控制该行为的实际执行。

通俗来说，就是我想点份外卖，但是手机没电了，于是我让同学用他手机帮我点外卖。在这个过程中，其实就是我同学（代理对象）帮我（被代理的对象）代理了点外卖（被代理的行为），在这个过程中，同学可以完全控制点外卖的店铺、使用的APP，甚至把外卖直接吃了都行（对行为的完全控制）。

 

通过以上类比，可以总结一下代理的 4 个要素：

  • 代理对象

  • 被代理的行为

  • 被代理的对象

  • 行为的完全控制

 

从实际编码的角度来说，假设遇到了与上面相同的需求，需要记录下一些方法的执行时间。最简单的方式当然就是在方法的开头记录一个时间戳，在 return 之前记录一个时间戳。

 

但如果方法的流程很复杂，需要在每一个 return 前都增加一行记录时间戳的代码，很麻烦。于是想到可以由方法的调用者来记录时间。

 

但是又会遇到另外一个问题，如果该方法在很多地方调用，或者需要记录的方法有多个，那么依然会面临重复手动写 log 代码的问题。

详细解释动态代理的工作流程

  1. 创建 InvocationHandler 实现类： mytest 类实现了 InvocationHandler 接口，在其 invoke 方法中定义了代理逻辑。

  2. 创建代理对象： 使用 Proxy.newProxyInstance 方法创建代理对象。此方法需要三个参数：

    • 类加载器 (ClassLoader)：用于加载代理类，可以使用目标类的类加载器。

    • 接口数组 (Class<?>[])：代理类需要实现的接口列表。

    • 调用处理器 (InvocationHandler)：实现了 InvocationHandler 接口的实例。

  3. 方法调用： 当调用代理对象的方法时，实际会调用 InvocationHandler 实现类的 invoke 方法。在这个方法内部，可以在调用目标对象的方法之前和之后添加自定义的逻辑。

 

代理对象方法调用流程

  • 当 proxy.doSomething() 被调用时，代理对象会将该调用委托给 InvocationHandler 的 invoke 方法。

  • 在 invoke 方法中：

    • 打印 "Logging before method execution"。

    • 打印开始时间。

    • 调用 method.invoke(new RealObject(), args) 执行 RealObject 的实际方法。

    • 打印结束时间。

    • 打印 "Logging after method execution"。

 

关于 invoke 方法的三个参数

在 Java 动态代理机制中，InvocationHandler 的 invoke 方法的三个参数分别是：

  1. Object proxy：这是代理对象本身。

  2. Method method：这是被调用的方法对象，表示代理对象上被调用的方法。

  3. Object[] args：这是调用方法时传递的参数数组。

 

参数传递过程

当调用代理对象的方法时，JVM 会自动处理这三个参数，并将它们传递给 InvocationHandler 的 invoke 方法。

具体过程如下：

  1. 代理对象创建： 使用 Proxy.newProxyInstance 方法创建代理对象时，指定了接口、类加载器和调用处理器（即实现了 InvocationHandler 接口的实例，在这里是mytest类的实例handler）。

  3. 参数传递： 在代理对象内部，JVM 会将以下信息传递给 invoke 方法：

    • proxy 参数：即当前代理对象 proxy 本身。

    • method 参数：一个 Method 对象，表示代理对象上被调用的方法。在这个例子中，它表示 SomeInterface 中的 doSomething 方法。

    • args 参数：调用该方法时传递的参数列表。在这个例子中，因为 doSomething 方法没有参数，所以 args 数组为空。

如果接口类中有多个方法，JVM 处理这些方法调用时的机制是相同的。每当通过代理对象调用任意方法时，JVM 都会将对应的方法对象 (Method)、参数列表 (Object[] args)，以及代理对象自身 (proxy) 传递给 invoke 方法。

处理流程

    • proxy 是代理对象。

    • method 是 doSomething 方法对象。

关于 invoke 参数的小结

  • Object proxy：代理对象本身。

  • Method method：表示被调用的方法对象。

  • Object[] args：调用方法时传递的参数数组。

总结

Java 动态代理机制通过 Proxy 类和 InvocationHandler 接口，使得在运行时创建代理对象并动态处理方法调用变得非常灵活和简单。这种机制非常适用于在不修改原有代码的情况下，增强方法调用的功能，比如日志记录、权限控制、事务管理等。
