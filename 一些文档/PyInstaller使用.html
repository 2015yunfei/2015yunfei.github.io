PyInstaller简介

PyInstaller 是一个用于将 Python 程序转换为独立可执行文件（在 Windows, Linux, Mac OS X 上）的工具，它能够将 Python 程序及其所有依赖项打包成一个单一的文件夹或可执行文件中，从而无需安装 Python 解释器即可运行。

 

如何使用PyInstaller

安装 PyInstaller:

在命令行中运行以下命令来安装 PyInstaller:

1
pip install pyinstaller

 

生成可执行文件:

打开命令行工具，切换到的脚本所在的目录，然后运行以下命令：

xxxxxxxxxx
1
1
pyinstaller yourscript.py

 

使用生成的可执行文件:

在默认情况下，PyInstaller 会在脚本的目录下创建一个 dist 文件夹，里面包含生成的可执行文件。在 Windows 上，这个文件通常会有 .exe 扩展名。双击该文件即可运行完成打包的程序。

 

可选参数:

PyInstaller 提供了许多可选参数来定制生成的可执行文件。

例如，可以使用 --onefile 参数来生成单个可执行文件，而不是一个包含依赖的文件夹：

xxxxxxxxxx
1
1
pyinstaller --onefile yourscript.py

 

关于依赖的处理:

如果的脚本依赖于外部库或者资源文件，PyInstaller 通常能够自动找到它们。如果 PyInstaller 不能找到所有的依赖，可能需要使用 --add-data、--add-binary 或者 --hidden-import 参数手动指定它们。

 

生成.spec 文件:

在某些复杂情况下，可能需要创建一个 .spec 文件来更详细地控制打包过程。

可以通过运行以下命令来生成一个基本的 .spec 文件：

xxxxxxxxxx
1
1
pyinstaller --onefile yourscript.py

然后可以编辑生成的 yourscript.spec 文件来调整打包过程。

 

调试问题:

如果在打包过程中遇到问题，PyInstaller 提供了 -v 或 --log-level=DEBUG 参数来获取更详细的输出，帮助诊断问题。

请注意，PyInstaller 打包后的可执行文件可能比原始的 Python 脚本大，因为它包含了 Python 解释器和所有依赖库。

另外，打包过程可能会因为不同操作系统的环境差异而有所不同，需要根据实际情况调整。

 

关于spec文件更细致的讨论

PyInstaller 在打包 Python 程序时，会生成一个 .spec 文件（例如 yourscript.spec），这个文件是一个配置文件，它包含了 PyInstaller 打包过程中所有的设置和选项。.spec 文件是一个 Python 脚本，它定义了哪些文件应该被包含在最终的可执行文件中，如何处理依赖项，以及其他打包相关的细节。

 

.spec 文件的主要作用包括：

  1. 自动化打包过程: 一旦生成了 .spec 文件，可以直接使用这个文件来打包的程序，而不是每次都使用命令行参数。这使得打包过程可以更容易地自动化，例如在持续集成系统中。

  2. 自定义打包配置: .spec 文件允许更细粒度地控制打包过程。例如，可以指定额外的数据文件、二进制文件、隐藏的导入（hidden imports），或者修改打包的选项。

  3. 解决复杂的依赖问题: 对于复杂的程序，PyInstaller 可能无法自动检测到所有的依赖。在这种情况下，可以编辑 .spec 文件，手动添加缺失的依赖项。

  4. 优化打包结果: 通过调整 .spec 文件中的选项，可以优化最终的可执行文件，例如减少文件大小，或者提高程序的启动速度。

  5. 跨平台打包: 如果需要在多个平台上打包同一个程序，.spec 文件可以帮助保持打包配置的一致性。

 

在生成 .spec 文件后，可以通过以下命令使用它来打包的程序：

xxxxxxxxxx
1
1
pyinstaller yourscript.spec

如果需要对 .spec 文件进行修改，可以使用任何文本编辑器打开它。.spec 文件是一个标准的 Python 脚本，其中包含了一些 PyInstaller 特定的类和函数。在修改 .spec 文件时，可以添加或修改类的实例，例如 Analysis、PYZ、EXE 等，以调整打包的细节。

 

关于打包图形化界面程序

如果想要打包后的程序在运行时不显示命令行窗口（在 Windows 上通常是 cmd.exe），可以使用 PyInstaller 的 --windowed 参数。这个参数会告诉 PyInstaller 将程序打包成一个窗口应用程序，而不是命令行应用程序。

 

在命令行中，使用以下命令来打包具有图形化界面的程序：

这将会生成一个没有命令行窗口的可执行文件。如果的程序是一个图形用户界面（GUI）应用程序，这将是非常有用的。

常见的PyInstaller参数

以下是一些常用的参数及其解释：

  1. --onefile: 这个参数告诉 PyInstaller 将所有文件打包成一个单一的可执行文件。如果不使用这个参数，PyInstaller 默认会创建一个包含依赖项的文件夹。

  2. --windowed: 使用这个参数会创建一个没有命令行窗口的窗口应用程序。这对于图形用户界面（GUI）应用程序非常有用。

  3. --add-data <SRC;DEST or SRC:DEST>: 这个参数用于包括额外的非二进制文件（如数据文件、图片等）。<SRC;DEST> 或 <SRC:DEST> 表示源文件和目标目录的路径，其中 SRC 是文件或目录的路径，DEST 是最终可执行文件内的目标路径。

  4. --add-binary <SRC;DEST or SRC:DEST>: 类似于 --add-data，但用于包括二进制文件。<SRC;DEST> 或 <SRC:DEST> 的含义与 --add-data 相同。

  5. --hidden-import <MODULENAME>: 如果 PyInstaller 没有自动检测到所有的依赖模块，可以使用这个参数手动添加它们。

  6. --exclude-module <MODULENAME>: 这个参数用于排除特定的模块，防止它们被包括在打包的应用程序中。

  7. --icon <ICONFILE>: 使用这个参数可以为打包的可执行文件指定一个图标。<ICONFILE> 是图标文件的路径。

  8. --upx-dir <UPX_DIR>: 指定 UPX（一个可执行文件压缩工具）的路径。如果 PyInstaller 没有自动找到 UPX，可以使用这个参数。

  9. --upx-exclude <FILE>: 指定不使用 UPX 压缩的文件。

  10. --distpath <DIR>: 指定生成的可执行文件和依赖项的存放目录。

  11. --workpath <DIR>: 指定临时工作目录的路径。

  12. --specpath <DIR>: 指定 .spec 文件的存放目录。

  13. --clean: 在构建之前清理 PyInstaller 的缓存和生成的文件。

  14. --log-level <LEVEL>: 设置日志级别。<LEVEL> 可以是 DEBUG, INFO, WARNING, ERROR 或 CRITICAL。

  15. --noconfirm: 在构建过程中跳过所有确认提示。

  16. --runtime-hook <RUNTIME_HOOKS>: 指定一个或多个 Python 脚本作为运行时钩子。这些脚本会在打包的应用程序启动时被导入。

 

这些参数可以根据的具体需求进行组合使用。例如，如果想创建一个单一的可执行文件，没有命令行窗口，并且包括一些额外的数据文件，可以使用以下命令：

xxxxxxxxxx
1
1
pyinstaller --onefile --windowed --add-data 'path/to/data/files;data' yourscript.py

每个参数的具体作用可能会随着 PyInstaller 版本的不同而有所变化，因此最好查阅所使用的 PyInstaller 版本的官方文档以获取最准确的信息。
